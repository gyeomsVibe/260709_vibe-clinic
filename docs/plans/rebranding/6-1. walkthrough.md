# 🩺 Vibe Clinic: 대시보드 동적 프로젝트 전환 기능 구현 완료 보고

웹 대시보드 화면상에서 현재 Vibe Clinic 프로젝트 본체와 하위 예제 프로젝트들(또는 임의의 다른 개발 디렉토리)을 실시간으로 선택 및 핫스왑(Hot-swap)할 수 있도록 고도화 작업을 무결하게 완료하였습니다.

---

## 1. 주요 변경 사항 (Accomplished Changes)

### 📁 백엔드 고도화 ([src/dashboard.js](file:///d:/D_Workspace_NB/-google-workspace/-antigravity-workspace/260709_vibe-clinic/src/dashboard.js))
- **동적 경로 변수 바인딩**: `startDashboard` 실행 중 기준 디렉토리를 `currentProjectDir` 상태 값으로 래핑하여 모든 API(진단 조회, 실행, 에러 패턴 로드, BYOK 설정 저장, AI 복구 등)가 실시간 동적 경로를 참조하도록 재설계했습니다.
- **신규 API 엔드포인트 수혈**:
  - `GET /api/project/list`: 코어 프로젝트 본체와 `examples/` 아래의 모든 예제 디렉토리 목록을 스캔하여 반환합니다.
  - `POST /api/project/change`: 프론트엔드에서 전송받은 경로로 `currentProjectDir`을 교체하고 임시 실행 결과 캐시를 리셋합니다.

### 📁 프론트엔드 고도화 ([src/dashboard.html](file:///d:/D_Workspace_NB/-google-workspace/-antigravity-workspace/260709_vibe-clinic/src/dashboard.html))
- **인터랙티브 프로젝트 선택 바 (Project Selector Bar) 신설**:
  - 대시보드 상단 로고 바로 밑에 프로젝트 선택 드롭다운과 절대 경로 입력창 및 **[이동]** 버튼을 조합한 프로젝트 컨트롤 영역을 배치했습니다. (BYOK 바의 세련된 다크모드 글래스모피즘 디자인 컴포넌트 재사용)
- **동적 리랜더링 파이프라인**:
  - 드롭다운 값을 바꾸거나 임의 경로 입력 후 이동 버튼을 누르면 `POST /api/project/change` API 통신이 작동합니다.
  - 성공적으로 프로젝트가 전환되면, 대시보드는 현재 프로젝트 명칭을 갱신하고 진단 항목 리스트(`fetchList`), 에러 패턴 아카이브(`loadErrors`), AI 설정 정보(`loadByokConfig`)를 실시간으로 다시 로딩하여 화면을 리프레시(Refresh)합니다.

### ② 사용 확인 가이드 (윤겸스 확인 방법)
바탕화면에 있는 **Vibe Clinic Dashboard** 바로가기 단축아이콘을 더블클릭하여 확인해보실 수 있습니다.

1. 바탕화면의 바로가기를 더블클릭합니다. (콘솔 창과 함께 [http://localhost:7700](http://localhost:7700) 웹 브라우저가 열립니다.)
2. 상단의 **📁 프로젝트 선택** 바에서 드롭다운(선택 콤보박스)을 조작합니다.
   - **Vibe Clinic 본체** 선택 시 ➔ `task-001-runner` 진단 카드 노출
   - **예제: calculator** 선택 시 ➔ `task-001-arithmetic`, `task-002-division-zero` 등 계산기 예제 카드 3개로 즉시 교체 노출
3. **직접 경로 입력** 테스트:
   - 만약 디바이스 상의 또 다른 Vibe Clinic 연동 프로젝트가 있다면, 해당 프로젝트의 절대 경로(예: `D:\workspace\my-app`)를 입력창에 적고 **[이동]**을 누르면 대시보드가 즉시 해당 타겟으로 경로를 변경하여 자가진단을 로드합니다.
