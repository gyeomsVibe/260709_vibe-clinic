# Vibe Clinic: 참조 보강형 현대식 폴더 선택 다이얼로그 구현 계획

`.NET` 순정 `FolderBrowserDialog`는 일부 환경에서 현대적 테마 승격을 무시하고 계속해서 구식 트리뷰("폴더 찾아보기") 창으로 고정 출력되는 문제가 잔존했습니다.

이를 근본적으로 극복하기 위해, 다시 Windows 현대식 탐색기 파일 오픈 인터페이스인 `IFileOpenDialog` C# 래핑 코드를 복원합니다. 

이전 컴파일 실패(CompilerError) 원인이었던 참조 라이브러리 누락을 막기 위해 **어셈블리를 명시적 다중 주입(`-ReferencedAssemblies @("System.Windows.Forms", "System")`)**하고, 스레드 아파트먼트 모델(`-STA`)을 지정해 100% 안정적으로 탐색기형 폴더 선택 팝업을 성공시킵니다.

---

## 1. 정밀 조치 및 변경 사항

### ① 백엔드 파워셸 다중 어셈블리 참조 및 EncodedCommand 교체 ([src/dashboard.js](file:///d:/D_Workspace_NB/-google-workspace/-antigravity-workspace/260709_vibe-clinic/src/dashboard.js))
- `/api/project/select` API의 파워셸 코드를 다중 어셈블리 참조가 결합된 `IFileOpenDialog` 래퍼로 교체합니다.
- C# 런타임 컴파일러가 `IntPtr` 및 `ComImport` 등 필수 형식을 정상 탐색하도록 `System`과 `System.Windows.Forms` 어셈블리를 명시적 바인딩합니다.
- Base64 UTF-16LE 인코딩과 `-STA` 스레드 옵션을 결합해 무결 기동합니다.
  ```powershell
  Add-Type -TypeDefinition $code -ReferencedAssemblies @("System.Windows.Forms", "System")
  ```

## 2. 검증 계획 (Verification Plan)

### 시나리오 테스트
1. **컴파일 에러 부재 확인**: `[📁 폴더 선택]` 클릭 시 붉은색 파워셸 컴파일 크래시 에러 로그 없이 즉각 기동되는지 확인.
2. **현대식 폴더 브라우저 기동 검증**: 팝업된 창이 구식 트리뷰("폴더 찾아보기")가 아닌, 완벽하게 상단 주소창과 우측 폴더 목록이 노출되는 현대식 파일 탐색기형 창인지 확인.
3. **더블클릭 진입 제한 확인**: 하위 폴더로 진입하지 않고 마우스 클릭 한 번 후 [폴더 선택] 버튼 클릭 시 정상적으로 창이 닫히며 경로가 전환되는지 확인.
