# Vibe Clinic: 폴더 다이얼로그 현대화, 에러 패턴 정합성 수정 및 프리미엄 로그/코드 디프 뷰어 구현 계획

Windows 현대식 탐색기 UI 기반의 폴더 선택 기능 구현, 프로젝트 전환에 따른 에러 패턴 목록 동기화 결함 조치, 그리고 진단 실행 로그 및 AI 치료 전/후의 코드 차이(Diff)를 아름답게 시각화해 주는 프리미엄 대시보드 UX/UI를 추가합니다.

---

## 1. 주요 고도화 설계 및 변경 사항

### ① Windows 현대식 파일 탐색기형 폴더 선택 대화상자 구현
- **백엔드 리팩토링 ([src/dashboard.js](file:///d:/D_Workspace_NB/-google-workspace/-antigravity-workspace/260709_vibe-clinic/src/dashboard.js))**:
  - 구형 트리 방식의 `FolderBrowserDialog`를 제거합니다.
  - Windows의 현대적 파일 탐색기 형태인 `OpenFileDialog`를 호출하여, 확인 버튼을 눌렀을 때 해당 파일의 상위 폴더 경로(`[System.IO.Path]::GetDirectoryName($dialog.FileName)`)를 동적으로 가로채 절대 경로를 도출해내는 스크립트 가드를 적용합니다.
    ```powershell
    $dialog = New-Object System.Windows.Forms.OpenFileDialog;
    $dialog.ValidateNames = $false;
    $dialog.CheckFileExists = $false;
    $dialog.CheckPathExists = $true;
    $dialog.FileName = '폴더 선택';
    $dialog.Title = 'Vibe Clinic 검증 대상 폴더를 선택하세요';
    if ($dialog.ShowDialog() -eq 'OK') { Write-Host ([System.IO.Path]::GetDirectoryName($dialog.FileName)) }
    ```

### ② 에러 패턴 목록 동기화 결함 조치
- **백엔드-프론트엔드 동기화 바인딩**:
  - 프로젝트 전환(`changeProject`) 시, 에러 패턴 캐시를 비우고 목록 리로드 함수(`loadErrors()`)가 새로운 `currentProjectDir` 경로에서 동작하는 물리적 비동기 순서를 확정 보장합니다.
  - 본체 프로젝트(`.`) 및 계산기 예제(`examples/calculator`) 아래의 실제 에러 패턴 파일 목록을 각각 다르게 렌더링되게 만듭니다.

### ③ 진단 실행 에러 로그 (Terminal Log View) 탑재
- **진단 카드 아코디언 확장 UI**:
  - 진단 카드가 실패(`ERROR` / `WARNING`)했을 때, 카드를 클릭하면 하단으로 콘솔 느낌의 터미널 테마 창(검은 배경, 모노스페이스 폰트, 네온 컬러 폰트)이 펼쳐집니다.
  - 진단 수행 결과 수집된 구체적인 스택 트레이스(Stack Trace) 및 `details` 데이터가 로그 뷰에 실시간 렌더링됩니다.

### ④ AI 치료(교정) 로그 및 코드 디프(Code Diff View) 뷰어 탑재
- **백엔드 치료 응답 스키마 확장 (`/api/repair`)**:
  - AI 복구 성공 시, 원본 코드(`originalCode`)와 수정본 코드(`repairedCode`), 그리고 변경 사항에 관한 AI 설명 피드백을 함께 반환하도록 `/api/repair` 응답 패킷을 확장합니다.
- **프론트엔드 코드 차이(Diff) 뷰어**:
  - 치료 완료 모달창에 **"AI 교정 리포트"** 영역을 배치합니다.
  - JavaScript 내장 텍스트 라인 비교(Line-by-line Diff) 알고리즘을 소형 실장하여, 수정 전(빨간색 배경/취소선) vs 수정 후(초록색 배경/추가) 코드 라인 차이를 깃허브 PR 스타일의 아름다운 뷰로 렌더링합니다.
