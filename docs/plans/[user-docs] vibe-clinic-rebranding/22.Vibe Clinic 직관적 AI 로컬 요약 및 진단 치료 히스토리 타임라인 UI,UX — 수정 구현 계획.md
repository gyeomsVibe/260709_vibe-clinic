# Vibe Clinic 직관적 AI 로컬 요약 및 진단 치료 히스토리 타임라인 UI/UX — 수정 구현 계획

20번 기획서(Opportunity Brief)의 3대 설계(로컬 폴백 요약, 치료 타임라인, 온보딩 가이드)는 코드에 반영되어 있으나, **실제 런타임에서 타임라인이 소실되는 치명적 버그**, **로컬 폴백 배지가 프론트엔드에 표시되지 않는 문제**, **수리 적용 후 드로어 동기화 누락** 등이 남아 있습니다. 이 수정 계획은 기존 구현의 실질적 오류를 모두 교정하고 완성도를 끌어올립니다.

---

## User Review Required

> [!CAUTION]
> `applyRepairProposal()` 함수(dashboard.html L1391–1402)에서 `repairStates[diagId]`를 **문자열**(`'repaired'`, `'failed:...'`)로 덮어쓰고 있습니다. 이로 인해 `addDiagnosticHistory()`가 축적한 `{ status, history: [...] }` 객체가 통째로 파괴되어 **치료 타임라인이 소실**됩니다. 이 버그는 수리 승인 직후 사이드 드로어를 열면 타임라인이 비어 있는 현상으로 나타납니다.

> [!WARNING]
> `cancelRepairProposal()`(L1411–1418)도 `delete repairStates[...]`로 히스토리를 통째로 삭제합니다. 취소 시 히스토리는 보존하고 status만 idle로 되돌려야 합니다.

> [!IMPORTANT]
> 로컬 폴백(Heuristic Fallback) 요약이 성공해도 프론트엔드 `loadProjectExplanation()`에서 `isFallback` 플래그를 확인하지 않아, **로컬 분석 대체 배지**가 전혀 표시되지 않습니다.

```diff
+ if (data.success) {
+   addDiagnosticHistory(diagId, 'ok', '✅ 승인 → 패치 적용 완료, 재진단 통과 (OK)');
+ } else {
+   addDiagnosticHistory(diagId, 'repair', '⚠️ 패치 적용됨, 재진단 아직 실패');
+ }

- repairStates[diagId] = 'failed:' + err.message;
+ if (!repairStates[diagId]) repairStates[diagId] = {};
+ repairStates[diagId].status = 'failed:' + err.message;
+ addDiagnosticHistory(diagId, 'repair', '❌ 수리 적용 요청 네트워크 실패: ' + err.message);
```

동일 함수 내 총 3곳의 문자열 할당을 모두 객체 속성 할당으로 교정합니다.
