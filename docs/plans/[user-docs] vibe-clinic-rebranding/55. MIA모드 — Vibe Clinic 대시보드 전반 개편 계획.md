## 1. Opportunity Brief (기획)

### 1.1 결정 사항

Vibe Clinic 대시보드를 **청색·적색 생명의 별 + 흰색 청진기**의 최종 브랜드 체계와, 코드 진단·치료 업무에 맞는 관제 화면으로 개편한다.

1. 기존 보라/시안 V 마크와 청진기 이모지 헤더를 제거한다.
2. 최종 결정안의 중앙 뱀 위치를 흰색 청진기로 치환한 두 브랜드 마크를 사용한다.
3. 첨부4의 정보 구조(사이드바, 핵심 게이지, 실시간 분석, 오류 피드, 디버그)를 채택하되, 실제 API가 제공하는 수치만 표시한다.
4. 단일 1,336줄 `App.jsx`를 기능 단위로 분리해 디자인 변경과 진단·치료 로직을 분리한다.
5. 변경 전 상태는 Git 작업공간 밖의 스냅샷과 단일 되돌리기 커밋 구조로 보존한다. `.bak` 파일은 사용하지 않는다.

### 1.2 최종 브랜드 규칙

| 구분 | 확정 규칙 | 사용 위치 |
| --- | --- | --- |
| 기본 마크 | 딥 네이비 생명의 별 + 흰색 청진기 | 일반 헤더, 프로젝트 선택, 기본 상태, 문서 표지 |
| 긴급 마크 | 레드 생명의 별 + 흰색 청진기 | 치료 시급, 긴급 치료 확인, 심각 오류·롤백 안내 |
| 청진기 | 중앙 지팡이를 따라 상승하는 흰색 청진기 | 두 마크에서 동일한 형상·비율 유지 |
| 금지 요소 | V, 방패, 체크마크, 네온 글로우, 보안 제품 인상 | 전체 대시보드 및 확장 헤더 |

`assets/[최종] 결정안.png`은 두 마크가 함께 들어 있는 **시각 기준 원본**이다. 실제 앱에는 이 이미지를 통째로 넣지 않고, 청색·적색 마크를 각각 분리한 정식 앱 자산으로 사용한다.

### 2.2 결정

**Go — 대안 C(단계적 재구성)**

- 디자인은 첨부4의 관제형 구조를 적용한다.
- 기능은 현재 API 계약과 안전 흐름을 유지한다.
- 브랜드는 최종 결정안의 청색 기본 마크와 적색 긴급 마크로 고정한다.
- 대시보드 코드만 별도 작업공간에서 수정하고, 검증 후 하나의 논리적 변경 단위로 병합한다.

### 2.3 생명의 별 표식 리스크

미국 NHTSA는 생명의 별(Star of Life)을 EMS 식별용 등록 인증표장으로 설명하며, 사용 전 문의를 안내한다. 이 계획은 법률 자문이 아니며, **공개 배포·상용 로고 교체·VSIX 재배포 전에는 표식 사용 가능 여부를 별도로 검토**해야 한다. 로컬 대시보드 개편과 목업 검증은 진행할 수 있으나, 외부 배포는 이 확인 전까지 보류한다.

## 4. Delivery Card (실행 세부 계획)

### Phase 0. 기준선 동결과 완전 롤백 준비

1. 현재 작업 트리와 사용자 변경 파일을 목록화한다.
2. 저장소 밖의 타임스탬프 폴더에 다음을 보관한다.
   - `dashboard-ui/`, `src/dashboard.js`, `src/dist/` 스냅샷
   - Git diff, 파일 SHA-256 목록, 현재 테스트·린트 결과
3. 별도 `ui/dashboard-v2` 작업공간에서 변경한다. 메인 작업공간은 건드리지 않는다.
4. 최종 병합은 한 개의 논리적 커밋으로 만들고, 되돌릴 때는 `git revert`만 사용한다.
5. 병합 전 별도 작업공간에서 복원 시험을 실행해 신규 파일·빌드 산출물까지 함께 제거되는지 확인한다.

### Phase 4. 데이터 패널과 치료 흐름

### Phase 5. 접근성·성능·정리

1. 모달에 `role="dialog"`, 포커스 이동·복귀, ESC 닫기, 배경 클릭 정책을 적용한다.

3. 1440px, 1024px, 768px, 390px에서 사이드바·드로어·패널 순서를 검증한다.

5. 실제 참조가 없는 `App.css`, Vite 기본 자산, 미사용 import·상태를 마지막에 제거한다.
6. 린트 경고를 0건으로 만든다.

## 5. 승인 게이트

### 지금 확정된 것

- 최종 브랜드: 청색 기본판 + 적색 긴급판, 두 판 모두 흰색 청진기
- 대시보드 정보 구조: 관제형 사이드바·실제 지표 게이지·진단/치료/로그 패널
- 구현 방식: 기능 동결 후 단계적 재구성
- 롤백 방식: 외부 스냅샷 + 분리 작업공간 + 단일 되돌리기 커밋

### 실행 전 필요한 승인

1. 외부 백업 폴더와 분리 작업공간 생성
2. 최종 결정안에서 청색·적색 앱 마크 두 개를 분리해 프로젝트 자산으로 추가
3. `dashboard-ui` 소스·CSS·빌드 산출물 변경
4. 공개 배포 전 생명의 별 표식 사용 검토 완료

이 계획은 구현 준비를 위한 `DIRECTION_DRAFT`다. 실제 대시보드 동작·브랜드 표식 사용 가능성은 코드 검증과 별도 공개 사용 검토가 끝난 뒤에만 `VERIFIED_RESULT`로 올린다.
