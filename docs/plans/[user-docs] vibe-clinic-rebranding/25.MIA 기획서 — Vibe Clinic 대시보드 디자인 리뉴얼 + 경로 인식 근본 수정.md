## 1. Opportunity Brief

### 해결하려는 문제

#### 디자인 문제: B&W 미니멀의 한계
현재 `#09090b` 배경 + `#fafafa` 텍스트의 순수 흑백 컬러 시스템은:
- "클리닉"이라는 의료·치료 브랜드 정체성을 전혀 전달하지 못함
- 상태(OK/WARN/ERROR) 구분이 흑백 명도 차이에만 의존해 가시성 낮음
- 좌측 패널(400px)에 3개 섹션이 밀집되어 답답하고 밸런스가 무너져 있음

#### 기능 문제: 한국어 경로 인코딩 깨짐 + 프로젝트 목록 제한
- `<option value="${p.path}">` HTML 보간으로 `(계산기)` → `(◆◆◆)` 인코딩 깨짐
- `sendJson` charset=utf-8 누락
- 프로젝트 목록이 `examples/`만 나열 → 워크스페이스 형제 38개 폴더 접근 불가
- AI 스캐너에 구현 기술 방식(How it's built) 설명 부재

### 핵심 가설
"클리닉"에서 연상되는 **메디컬 티얼/사이안** 색조를 시그니처 컬러로, 상태별 유의미한 색상 분리를 적용하고, 공간 비율을 재설계하면 가독성·가시성·브랜드 신뢰감이 동시에 올라간다.

### 🏗️ 공간 밸런스 재설계

**현재 문제점:**
- 좌측 400px에 프로젝트 선택기 + BYOK + AI 스캐너 3개가 수직 밀집
- 모눈 도트 배경이 시각 노이즈 추가
- 카드 내부 padding이 작아 답답함

**변경안:**
1. **배경**: 모눈 도트 제거 → 부드러운 gradient mesh (`radial-gradient` 사용)
2. **좌측 패널**: `400px` → `380px`, 내부 패딩 `16px → 20px`, 각 섹션 간격 `16px → 20px`
3. **카드**: 패딩 `16px → 20px`, border-left 컬러를 상태별로 확실히 분리 (OK=에메랄드, WARN=앰버, ERR=로즈)
4. **버튼**: `진단 실행` 버튼을 **시그니처 티얼** 배경으로, 호버 시 글로우 효과
5. **Health Ring**: 링 색상을 상태에 따라 에메랄드/앰버/로즈로 동적 변경 (기존: 흰색 단일)
6. **상태 도트**: 실제 색상 반영 (현재: 흑백 명도 차이만)
