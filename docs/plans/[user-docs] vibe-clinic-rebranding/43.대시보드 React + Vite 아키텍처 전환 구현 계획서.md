## 3. 제안된 변경 사항 (Proposed Changes)

### 3.1 프론트엔드 새 프로젝트 구성
#### [NEW] `dashboard-ui/`
*   `npx create-vite dashboard-ui --template react --no-interactive` 명령으로 React 프로젝트를 생성합니다.
*   빌드 결과물이 기존 백엔드 경로인 `src/dist`로 출력되도록 `dashboard-ui/vite.config.js`를 커스텀 구성합니다.
*   Tailwind CSS 대신 순수 CSS/CSS Modules 및 글래스모피즘 변수 시스템을 적용하여 경량화 및 미려한 룩앤필을 구현합니다.

#### [NEW] `dashboard-ui/src/index.css` (글래스모피즘 & 에메랄드 그린 디자인 시스템)
*   에메랄드 그린(`--ok-emerald: #10b981`, `#059669`)을 메인 테마 포인트 컬러로 지정합니다.
*   `backdrop-filter: blur(25px)` 및 반투명 테두리(`rgba(255, 255, 255, 0.06)`)를 선언하여 완성도 높은 글래스모피즘 효과를 구현합니다.

#### [NEW] `dashboard-ui/src/App.jsx` & Components
*   **State Management:** 로컬 React `useState`/`useEffect`를 사용하되, API 요청에 대한 디바운스 유틸리티를 제작하여 불필요한 네트워크 리액션 및 컴포넌트 리렌더링을 방지합니다.
*   **Performance Optimization:**
    *   **Intersection Observer 가상 리스트:** 수십 개의 진단 카드 렌더링 시 뷰포트 내 카드만 온전하게 표시되도록 커스텀 React 훅(`useIntersectionObserver`) 구현.
    *   **로깅 청크 렌더러:** 대량의 터미널 텍스트 로딩 시 렌더링 지연을 해소하기 위해 `requestAnimationFrame` 스케줄러 기반 로더 컴포넌트 탑재.
