# Vibe Clinic (바이브 클리닉) 리브랜딩 구현 계획

이 계획은 `vibe-diagnosis` 프로젝트를 **`Vibe Clinic` (바이브 클리닉)**으로 개편하고, 모든 파일 구조, CLI 도구, MCP 서버, VS Code 확장프로그램의 브랜딩과 식별자를 완전히 변경하기 위한 상세 작업 절차입니다.

## 브랜드 명칭 정의

| 항목 | 기존 명칭 | 신규 명칭 (`Vibe Clinic`) |
| --- | --- | --- |
| **프로젝트명 (npm)** | `vibe-diagnosis` | `vibe-clinic` |
| **CLI 바이너리 명령어** | `vibe-diag` | `vibe-clinic` (단축 명령: `vbc`) |
| **설정 디렉토리** | `.vibe-diagnosis/` | `.vibe-clinic/` |
| **진단 스크립트 확장자** | `.diag.js` | `.clinic.js` |
| **MCP 서버명** | `vibe-diagnosis` | `vibe-clinic` |
| **MCP 도구(Tools)** | `run_diagnostics`, `init_diagnostics`, `list_diagnostics` | `run_clinic`, `init_clinic`, `list_clinics` |
| **VS Code 확장 ID** | `vibe-diagnosis-vscode` | `vibe-clinic-vscode` |
| **VS Code 표시명** | `Vibe Diagnosis` | `Vibe Clinic` |
| **VS Code 설정 네임스페이스** | `vibeDiagnosis` | `vibeClinic` (예: `vibeClinic.cliPath`) |
| **VS Code 명령어 접두사** | `vibeDiagnosis.*` | `vibeClinic.*` (예: `vibeClinic.run`) |

## 검증 계획

### 자동 테스트

1. 프로젝트 루트에서 `npm test`를 구동하여 바뀐 모듈 명칭 하에서도 모든 단위 테스트(11개)가 동일하게 통과하는지 확인합니다.
2. `vibe-clinic` 로컬 CLI를 통한 테스트:
   `node bin/vibe-clinic.js run` 명령어가 정상 동작하여 100% Health 결과를 출력하는지 검증합니다.

### 수동 검증

1. `node bin/vibe-clinic.js init` 명령어를 실행하여 새로운 `.vibe-clinic/` 설정 디렉토리가 의도한 템플릿 파일들과 함께 정상 스캐폴딩되는지 확인합니다.
2. 로컬 VS Code 확장프로그램 빌드 및 수동 로드 테스트를 통해, Antigravity IDE 상에서 `Vibe Clinic: Run` 등의 명령어가 정상 호출되는지 점검합니다.
