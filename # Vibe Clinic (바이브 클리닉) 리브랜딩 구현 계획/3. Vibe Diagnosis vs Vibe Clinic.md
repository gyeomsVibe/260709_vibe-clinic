# Vibe Diagnosis vs Vibe Clinic 비교 자료 검증 기록

## 1. 검증 목적

첨부된 비교 초안의 주장을 원작 저장소와 현재 Vibe Clinic 작업 트리에서 다시 확인하고, 사실로 확인된 개선점만 프로젝트에 반영한다.

- 원작 저장소: [Rejard/vibe-diagnosis](https://github.com/Rejard/vibe-diagnosis)
- 원작 확인 스냅샷: `1207e91606cbc7197fdc079cb89ec74f2f5ce38d`
- 원작 스냅샷 시각: 2026-07-14 09:05:04 +0900
- 원작 코어 패키지: `vibe-diagnosis@1.2.5`
- 원작 VS Code 확장: `vibe-diagnosis-vscode@1.2.6`
- 현재 저장소: `gyeomsVibe/260709_vibe-clinic`
- 현재 기준 커밋: `f546be03ee6e368520c72a1eb78531e0dba3ef0d`
- 현재 코어/MCP/VS Code 확장: 모두 `2.0.0`

> 버전은 같은 숫자로 묶지 않는다. 원작 코어는 `1.2.5`, 원작 확장은 `1.2.6`이며, Vibe Clinic은 하드 컷(Hard Cut) 이후 `2.0.0`이다.

## 3. 확인된 사실

### CLI 호출

- 원작 확장은 `child_process.exec`로 셸 명령 문자열을 실행한다.
- 원작 확장은 로컬 CLI를 찾지 못하면 `npx vibe-diag`를 시도한다.
- Vibe Clinic 확장은 `execFile`로 Node 실행 파일과 인자 배열을 직접 전달한다.
- Vibe Clinic 확장은 사용자 지정 `vibeClinic.cliPath`, 저장소 CLI, 워크스페이스 CLI, 프로젝트 로컬 패키지 순서로 탐색하며 `npx`로 폴백하지 않는다.

### 대시보드 프로세스

- 원작 확장은 대시보드도 `exec`와 5초 타임아웃으로 실행한다.
- Vibe Clinic 확장은 대시보드를 `spawn({ detached: true, stdio: 'ignore' })`와 `unref()`로 분리한다.
- 따라서 Vibe Clinic 쪽이 장기 실행 프로세스의 수명 주기를 명시적으로 분리한다. 실제 운영 환경별 지속 시간은 별도 통합 테스트가 필요하다.

### 에이전트 규칙

- Vibe Clinic은 `GEMINI.md`를 기준으로 프로젝트 내부 `.claude/skills/vibe-check/SKILL.md`를 동기화한다.
- 기본 `npm run sync:rules`는 프로젝트 파일만 변경한다.
- 사용자 홈의 Claude 스킬은 `npm run sync:rules:global`을 명시적으로 실행할 때만 변경한다.
- 이 구분은 프로젝트 작업이 사용자 전역 상태를 묵시적으로 덮어쓰지 않도록 하기 위한 안전 조치다.

### 리브랜딩 범위

- CLI, 설정 디렉터리, 진단 파일 확장자, MCP 도구명, VS Code 명령 ID가 Vibe Clinic 명칭으로 전환됐다.
- 기존 `.vibe-diagnosis/`와 `*.diag.js`를 읽는 호환 계층은 두지 않는다.
- Apache-2.0 라이선스와 원작자 기여 표기는 유지한다.

## 4. 초안에서 교정한 부분

- Vibe Clinic 버전 `1.1.4` 표기를 `2.0.0`으로 교정했다.
- 원작의 `1.2.6`은 VS Code 확장 버전이며 코어 패키지는 `1.2.5`임을 분리했다.
- “완벽히 해결”, “원천 차단”, “훨씬 고도화” 같은 검증 범위를 넘는 표현을 제거했다.
- 원작의 변경을 문서 수정으로만 단정하지 않고, 확인한 소스 스냅샷 범위로 한정했다.
- 마켓플레이스 배포 여부나 실제 사용자 환경의 안정성은 확인하지 않았으므로 결론 근거에서 제외했다.

## 5. 이번 반영 결정

- README의 최초 승인 문구를 복사 가능한 코드 블록으로 변경한다.
- 규칙 동기화는 프로젝트 로컬을 기본값으로 하고 전역 동기화는 별도 명령으로 분리한다.
- 비교 보고서에 원작 스냅샷, 버전 차이, 검증 한계를 명시한다.
- `package-lock.json` 두 개를 현재 `2.0.0` 메타데이터와 맞춘다.
- 현재 소스로 만든 `vibe-clinic-vscode-2.0.0.vsix`만 유지하고, 사용자 승인에 따라 구 `vibe-diagnosis-vscode-1.1.4.vsix`는 삭제했다.
- Apache-2.0 본문을 유지하고 `NOTICE`에 Vibe Clinic 수정 저작권과 원작자 귀속을 함께 기록했다.

## 6. 검증 한계

- VSIX 파일의 구조와 메타데이터는 확인했지만 VS Code GUI에 직접 설치하는 수동 테스트는 수행하지 않았다.
- npm 공개 배포, 마켓플레이스 배포, 릴리스 생성은 수행하지 않았다.
- 비교 결과는 위에 적은 두 저장소 스냅샷 기준이며 이후 원작 변경은 자동 반영되지 않는다.
