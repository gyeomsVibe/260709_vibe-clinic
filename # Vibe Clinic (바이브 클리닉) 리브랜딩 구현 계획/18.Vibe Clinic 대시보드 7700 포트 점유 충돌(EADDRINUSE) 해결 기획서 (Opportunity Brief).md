# Vibe Clinic 대시보드 7700 포트 점유 충돌(EADDRINUSE) 해결 기획서 (Opportunity Brief)

이 기획서는 윤겸스님이 대시보드 서버를 수동으로 띄울 때 발생하는 7700 포트 충돌(EADDRINUSE) 에러의 원인을 해결하고, 로컬 개발 환경을 정상 상태로 복구하기 위한 기획 및 설계안입니다.

---

## 1. Opportunity Brief (기획안)

### 해결하려는 문제 (Observable Problem)
- **7700 포트 점유 충돌**:
  - 에이전트 측에서 미니멀 캔버스 UI가 정상 빌드되었는지 테스트 검증하기 위해 백그라운드로 7700 포트 대시보드 서버(`task-369`)를 구동해 둔 상태였습니다.
  - 이로 인해 사용자가 직접 대시보드 기동 배치 파일(`run-dashboard.bat`)을 실행하거나 VS Code 상에서 대시보드를 구동하려고 할 때, 동일 포트인 `7700`이 이미 사용 중이라는 `EADDRINUSE` 에러가 발생하면서 기동이 실패하고 멈추는 오동작이 발생합니다.

### 대상 사용자 및 작업 정의 (Job-To-Be-Done)
- **작업**: 아무런 포트 충돌 방해 없이, 제공된 배치 파일(`run-dashboard.bat`)이나 단축키를 눌렀을 때 대시보드 서버가 7700 포트 상에서 정상적으로 실행되고 웹 브라우저가 원활하게 로드되기를 원함.

### 목표 성과 및 핵심 가설 (Desired Outcome & Hypothesis)
- **목표 성과**:
  - 에이전트의 백그라운드 대시보드 서버 프로세스(`task-369`)를 즉시 종료하여 7700 포트를 깨끗하게 개방.
  - 사용자가 다시 배치 파일 실행 시 아무 충돌 없이 대시보드 기동이 가능함을 보장.
- **핵심 가설**: 에이전트 측의 테스트용 백그라운드 서버를 확실하게 정리하여 로컬 포트를 반환하면, 사용자가 직접 실행하는 개발자 구동 흐름(배치 파일, VS Code 단축 명령어 등)이 충돌 없이 원활히 정상 복구될 것이다.

---

## 2. User Review Required (피드백 요청 사항)

### 포트 반환 및 사용자 직접 실행 권고
- 에이전트가 점유하고 있는 백그라운드 7700 포트 서버를 즉시 `kill` 종료하겠습니다.
- 그 후 사용자의 기동 창(`계속하려면 아무 키나 누르십시오 . . .`)에서 아무 키나 입력하여 창을 닫으신 뒤, `run-dashboard.bat`을 다시 기동하여 최신 개편된 대시보드를 안전하게 띄우는 흐름을 제안합니다.

---

## 3. Proposed Changes (변경 제안 내용)

- 소스 코드 상의 추가 변경은 없으며, 백그라운드 구동 중인 프로세스 정리만 수행합니다.

### 1) 백그라운드 태스크 종료
- `task-369` (7700 포트 구동 서버) 프로세스를 즉시 `kill` 처리합니다.

---

## 4. Verification Plan (검증 및 테스트 계획)
- 에이전트 측 백그라운드 태스크 목록을 조회하여 포트 7700을 점유하는 서버가 완전히 비워졌는지 확인합니다.
