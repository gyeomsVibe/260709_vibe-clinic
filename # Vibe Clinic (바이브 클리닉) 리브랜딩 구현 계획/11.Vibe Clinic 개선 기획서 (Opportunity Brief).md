# Vibe Clinic 개선 기획서 (Opportunity Brief)

이 기획서는 `vibe-clinic` 자가 진단 프레임워크의 유용성과 사용성을 극대화하기 위한 추가 개선 작업을 기획하고 검토하는 문서입니다.

---

## Opportunity Brief

### 1. 해결하려는 문제 (Observable Problem)
현재 Vibe Clinic의 핵심 기능 중 하나인 **AI Auto Repair**는 실패한 진단을 자동으로 고쳐주지만, 다음과 같은 한계가 있습니다:
- **코드 덮어쓰기 위험**: AI가 수정한 코드가 기존 로컬 파일에 즉시 덮어씌워지며, 사용자가 변경 사항을 사전에 확인하거나 검토할 수 없습니다. 이로 인해 코드 유실이나 불안정한 코드가 삽입될 수 있는 위험이 있습니다.
- **개발 환경의 파편화**: VS Code 확장프로그램에서 대시보드를 실행할 때 외부 기본 웹 브라우저가 강제로 열려 개발 컨텍스트가 분산됩니다.
- **UI/UX 미흡**: 현재 대시보드의 에러 패턴 뷰어 및 수리 흐름 UI가 다소 투박하고 정보 밀도가 낮아 프리미엄 도구로서의 감성을 주기 어렵습니다.

### 2. 대상 사용자 및 작업 정의 (Stakeholder & Job-To-Be-Done)
- **대상**: AI 에이전트와 페어 프로그래밍을 하거나 자가 진단 기반의 코딩 작업을 수행하는 개발자.
- **작업**: 실패한 진단을 안전하고 시각적으로 신속하게 복구하며, 프로젝트의 전체적인 건강(Health) 상태를 한눈에 모니터링하고자 함.

### 3. 목표 성과 및 핵심 가설 (Desired Outcome & Hypothesis)
- **목표 성과**: AI 수리를 적용하기 전에 변경 사항을 시각적으로 안전하게 확인(Diff)하고 승인할 수 있는 프로세스를 제공하여 안전성을 보장하고, VS Code 통합 대시보드를 통해 개발 컨텍스트를 유지함.
- **핵심 가설**: AI의 코드 제안을 Diff 뷰어로 시각화하고 사용자 승인 절차를 도입하면, 자동 수리에 대한 개발자의 심리적 안정감이 비약적으로 상승하고 잘못된 자동 수리로 인한 롤백 빈도가 현저히 감소할 것이다.

## User Review Required

프로젝트 개선을 위한 3가지 실질적 대안을 제시합니다. 이 중 진행할 방향을 검토해 주시기 바랍니다.

### 후보 1. (추천) 대시보드 AI Auto Repair Diff Viewer 도입 및 프리미엄 UI/UX 고도화
대시보드 웹앱 내에서 Auto Repair 실행 시, AI가 수정한 코드를 즉시 덮어쓰지 않고 기존 코드와의 Diff(차이점)를 시각적으로 보여주는 **Diff Viewer**를 제공합니다. 사용자가 승인(`Approve & Apply`)해야만 파일에 반영됩니다. 추가로 대시보드 UI를 Glassmorphism 및 Sleek Dark Theme 기반의 프리미엄 디자인으로 업그레이드합니다.
- **장점**: 코드 유실 방지, 사용자 통제력 강화, 대시보드 완성도 향상.
- **단점**: 프론트엔드 코드(JS, CSS) 작업 범위가 넓어짐.

### 후보 2. VS Code 확장프로그램 내장 Webview 대시보드 통합
현재 외부 웹 브라우저에서 열리는 Vibe Clinic 대시보드를 VS Code 내부의 `Webview Panel`로 직접 렌더링하도록 VS Code 확장을 개선합니다.
- **장점**: 외부 브라우저 이동 없이 VS Code 내부에서 완벽한 원스톱 모니터링 가능.
- **단점**: VS Code Webview 보안 제약 및 통신 브릿지(IPC) 복잡성 증가.

### 후보 3. 지능형 진단 자동 추천 및 스마트 생성(Smart Scaffold)
프로젝트 소스코드를 분석하여 어울리는 진단(`*.clinic.js`)의 스켈레톤 코드를 AI가 분석하고 추천해 주는 생성 CLI/API를 탑재합니다.
- **장점**: 초기 도입 시 진단 작성의 귀찮음을 혁신적으로 해결.
- **단점**: 고품질의 진단 코드를 생성하기 위해 복잡한 프롬프트 엔지니어링 필요.
