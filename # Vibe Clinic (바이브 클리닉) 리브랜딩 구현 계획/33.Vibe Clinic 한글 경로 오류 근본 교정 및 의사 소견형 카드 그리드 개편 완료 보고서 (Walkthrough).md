# Vibe Clinic 한글 경로 오류 근본 교정 및 의사 소견형 카드 그리드 개편 완료 보고서 (Walkthrough)

## 1. 수정 및 개편 내역

### 📁 한글 경로 인식 오류 완전 해결 ("트래커_캘린더" 등)
- **PowerShell UTF-8 명시**: `folder-picker.ps1` 최상단에 `$OutputEncoding`과 `[Console]::OutputEncoding`을 강제로 UTF-8로 선언하여, Windows 한글 CMD(CP949)가 경로 스트림을 출력할 때 `T??L_K????` 등으로 문자가 깨지는 현상을 차단했습니다.
- **`readBody()` 버퍼 축적**: 백엔드에서 JSON 요청 데이터를 스트림으로 수집할 때 String 대신 Buffer 배열로 온전히 수집한 후 `Buffer.concat().toString('utf8')`로 결합함으로써, 멀티바이트 유실이나 청크 경계 오류를 완벽하게 차단했습니다.

### 🤖 AI 스캐너 무한 루프(Hang) 및 지연 제거
- **API fetch 12초 타임아웃**: `ai-provider.js`에 `AbortController`를 통한 12초 타임아웃을 적용해 네트워크 정체 시 무한 루프 스피너를 방지했습니다.
- **429/503 및 인증 오류 즉시 폴백**: 할당량 제한(429)이나 모델 지연(503) 또는 API 키 오류가 감지되면, 무모하게 대기하거나 재시도하지 않고 **즉시 지연 없이 로컬 스마트 정적 스캐너 결과로 폴백**합니다.
- **Gemini 단일화**: OpenAI, Anthropic 등 불필요한 제공자를 모두 제거하고 오직 **Gemini API 설정**만 기재하도록 BYOK를 간소화했습니다.

### 🩺 의사 소견형 3열 카드 그리드 및 공간 최적화 (목업 완벽 구현)
- **캔버스 삭제 및 반응형 그리드**: 마인드맵형 연결선 캔버스를 제거하고, 목업(Attachment 3)의 깔끔한 **3열 반응형 카드 그리드**로 개편하여 공간 활용도를 극대화했습니다.
- **필터 툴바 단일화**: `전체 검사`, `완치(정상) 🟢`, `주의 관찰 🟡`, `치료 시급 🔴` 한글 필터 버튼으로 교체하여 실시간 필터링을 제공합니다.
- **의사 소견 및 치료 비유 한글화**:
  - `SYSTEM / TASK / FUNCTION` -> `시스템 환경 검사`, `핵심 업무 흐름 검사`, `개별 기능 단위 검사`
  - `OK / WARNING / ERROR / PENDING` -> `완치 (정상 동작)`, `주의 관찰 필요 (경고)`, `치료 시급 (오류 발견)`, `진단 검사 대기`
  - 각 진단 카드 내에 **🩺 의사 소견 (Doctor's Note)**을 신설하여 환자 눈높이에서 검진 상태와 적절한 치료 방향(수리 처방)을 친절하게 제시합니다.

### 🐛 [NEW] 로컬 파일 프로토콜 및 웹뷰 403 Forbidden Origin 오류 해결
- **CORS/Origin 우회 허용**: 브라우저에서 `file:///` 프로토콜로 `dashboard.html`을 실행하거나 VS Code 웹뷰 환경에서 호출하는 경우, 브라우저가 전송하는 `Origin: null` 또는 `vscode-webview://` 헤더가 보안 검사(`isAllowedDashboardOrigin`)에 막혀 403 Forbidden 오류가 발생하고 "진단 실행에 실패했습니다." 토스트 팝업이 뜨는 문제를 해결했습니다.
  - `dashboard.js`에서 `null`, `vscode-webview://`, `vscode-file://`, `.vscode-cdn.net` 등의 로컬/웹뷰 기원의 요청을 안전하게 허용하도록 보안 필터를 수정했습니다.
- **로컬 파일 API 주소 폴백**: 브라우저의 프로토콜이 `file:`일 때 API 호출 주소가 `file:///api/run`과 같이 상대 경로로 꼬이지 않도록, 자동으로 `http://localhost:7700` 서버로 전달되도록 `dashboard.html` 내 `API` 변수를 개선했습니다.
- **에러 모니터링 세분화**: 진단 중 백엔드 에러가 발생한 경우 단순히 실패 토스트만 띄우지 않고, `data.error`에 담긴 정확한 상세 메시지를 사용자에게 알리도록 `runDiagnostics()`의 예외 처리 로직을 수정했습니다.

## 3. 수동 검수 가이드

1. **`run-dashboard.bat`**를 재실행해 7700 포트에 대시보드 서버를 기동합니다.
2. 브라우저로 **`http://localhost:7700`**에 직접 접속하거나, `dashboard.html` 파일을 로컬에서 더블 클릭해서 열어도 모두 에러 없이 정상적으로 데이터를 로드합니다.
3. 프로젝트가 변경되거나 진단 실행을 할 때 403 Origin 에러로 막히지 않고, 실시간으로 진단 결과 카드가 교체 및 갱신되는지 확인합니다.
