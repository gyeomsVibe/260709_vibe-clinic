# Vibe Clinic 직관적 AI 로컬 요약 및 진단 치료 히스토리 타임라인 UI/UX 기획서 (Opportunity Brief)

이 기획서는 Google Gemini API 503 에러로 인한 요약 실패 시 이를 지능적으로 복구해 주는 **로컬 휴리스틱 분석 요약(Local Heuristic Fallback)** 기능과, 사용자가 진단과 수리가 정상적으로 이루어졌는지 100% 직관적으로 파악할 수 있도록 돕는 **치료 히스토리 타임라인(Treatment History Timeline)** 및 **온보딩 가이드**를 추가 장착하는 대시보드 고도화 계획안입니다.

---

## 1. Opportunity Brief (기획안)

### 해결하려는 문제 (Observable Problem)
1. **지속적인 API 503 에러로 인한 요약 기능 마비**:
   - Google Gemini API 무료 티어 등에서 일시적인 트래픽 분산 지연(503)이 지속 발생하여, 3회 백오프 재시도 후에도 최종 실패하여 스캐너 패널이 에러 상태로 멈추는 상황입니다.
   - API 연결이 불안정하더라도 사용자는 최소한의 프로젝트 사양과 성격을 볼 수 있어야 합니다.
2. **진단 및 치료 과정의 블랙박스화 (직관성 및 신뢰성 부재)**:
   - 사용자는 대시보드에서 `진단 실행`과 `수리 적용`을 눌렀을 때, 코드가 실제로 어떻게 치료되어 최종 통과 상태가 되었는지 직관적으로 인지하기 힘듭니다. "과연 고쳐진 것이 맞나?" 하는 의문을 가질 수밖에 없습니다.

### 대상 사용자 및 작업 정의 (Job-To-Be-Done)
- **작업**: API가 장애 상태이더라도 내 프로젝트의 뼈대 요약을 상시 제공받고, 진단 카드가 실패했다가 AI 치료를 통해 성공(OK)으로 복구되는 전체 과정을 타임라인 이력과 온보딩 설명을 통해 투명하고 명확하게 시각적으로 이해하고 싶음.

### 목표 성과 및 핵심 가설 (Desired Outcome & Hypothesis)
- **목표 성과**:
  - API 최종 실패 시 작동하는 **로컬 정적 분석(Local Heuristic Fallback)** 엔진 장착.
  - 진단 카드와 사이드 인스펙터 패널 내에 **자가 진단 & 치료 타임라인(Treatment History Timeline)** 적용.
  - 대시보드 상단에 사용 방법을 한눈에 돕는 슬라이드형 **"🩺 Vibe Clinic 치료 가이드(온보딩)"** 위젯 배치.
- **핵심 가설**: API 에러 시에도 로컬 분석 요약을 제공하고, 복구가 완료된 진단 카드에 시간대별 치료 이력 타임라인을 투명하게 드러내면 기술 지식이 얕은 사용자도 시스템의 안정성을 실시간으로 확신하고 도구를 신뢰하게 될 것이다.

## 2. User Review Required (피드백 요청 사항)

이번 "투명한 자가 치료 및 로컬 요약" 구현을 위한 설계 사양을 검토해 주시기 바랍니다.

### 설계 A. AI 503 대응 로컬 휴리스틱 분석 (Local Fallback Summary)
- **동작**: 3회 지수 백오프 AI 호출 실패 시, 백엔드에서 에러를 뱉는 대신 프로젝트 메타데이터를 기반으로 로컬 휴리스틱 요약을 자동 생성하여 반환합니다.
- **요약 알고리즘**:
  - `package.json`의 `dependencies` 키워드 분석 (예: `next`, `react` 감지 시 "React/Next.js 기반 웹 애플리케이션", `express` 감지 시 "Express Node.js 백엔드 서버" 등).
  - 파일 확장자 통계 (예: JS 파일 10개, MD 파일 2개 등) 및 폴더 구조 스캔.
- **UI 표기**: 요약 패널 하단에 `[🤖 AI 엔진 일시 지연 - 로컬 스마트 분석 결과로 대체됨]` 배지를 띄워 정보 제공의 연속성을 확보합니다.

### 설계 B. 치료 히스토리 타임라인 (Treatment History Timeline)
- **타임라인 저장소 (`repairStates` 확장)**:
  - 각 진단 항목별로 `history` 배열을 브라우저 메모리에 누적 저장합니다.
  - 히스토리 상태 유형:
    1. `🔍 [시간] 진단 수행 -> 결과: ERROR (내용 요약)`
    2. `💡 [시간] AI 수리 제안 생성 완료`
    3. ` 승인 [시간] 윤겸스님 승인 -> 패치 적용 완료`
    4. `✨ [시간] 재진단 자동 수행 -> 결과: OK (복구 완료)`
- **UI 연동**:
  - 사이드 인스펙터 드로어(Drawer) 하단에 **"치료 이력 (History)"** 영역을 타임라인 디자인(점선 및 원형 아이콘 구조)으로 렌더링합니다.
  - 치료 완료(`repaired`)된 노드의 카드 하단에도 "치료 완료 이력보기" 단축 링크를 제공합니다.

### 설계 C. Vibe Clinic 온보딩 치료 가이드
- 대시보드 헤더 우측 혹은 중앙에 `[🩺 사용 가이드]` 토글 버튼 배치.
- 클릭 시 미니멀한 shadcn/ui 스타일의 설명 팝업이 뜨며, "1. 프로젝트 선택 -> 2. 진단 실행 -> 3. 수리 제안 검토 및 승인 -> 4. 완치 확인" 의 4단계 과정을 직관적인 일러스트성 이모지와 함께 노출합니다.
