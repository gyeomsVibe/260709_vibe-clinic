# Vibe Clinic 타 프로젝트 연동 및 미초기화 오류 근본 해결 기획서 (Opportunity Brief)

이 기획서는 대시보드 웹 UI에서 다른 프로젝트를 선택했을 때 진단이 수행되지 않거나 UI 통계와 진단 리스트 간 정합성이 어긋나는 오류를 근본적으로 해결하기 위한 기획 및 설계안입니다.

---

## 1. Opportunity Brief (기획안)

### 해결하려는 문제 (Observable Problem)
1. **UI 정합성 불일치**: 진단 파일(`.clinic.js`)이 존재하지 않는 프로젝트를 불러오는 경우, `runDiagnostics`는 가상 경고 노드 `_no_diagnostics`를 반환하므로 대시보드 상단 통계 바에는 `1 전체 / 1 경고(WARN)`로 표시됩니다. 하지만 진단 목록을 렌더링하는 `diagnosticsList`는 빈 배열(`[]`)이어서 화면 중앙에는 "진단 항목이 없습니다"로만 표시되어 사용자가 혼란을 겪습니다.
2. **웹 대시보드 내 초기화 수단 부재**: 대시보드에서 `.vibe-clinic` 폴더가 아예 없는 생소한 프로젝트를 가져왔을 때, CLI나 VS Code 확장을 쓰지 않고 브라우저 상에서 이를 즉시 초기화(Initialize)하여 활성화할 수 있는 기능이 없어 프로세스가 먹통이 됩니다.

### 대상 사용자 및 작업 정의 (Job-To-Be-Done)
- **작업**: 여러 프로젝트를 대시보드에서 전환하며 즉시 자가 진단을 수행하고, 진단 환경이 구축되지 않은 프로젝트는 즉각 대시보드 상에서 초기화하여 진단을 작동시키고 싶음.

### 목표 성과 및 핵심 가설 (Desired Outcome & Hypothesis)
- **목표 성과**: 진단 누락 상태를 명확히 안내하고, 웹 대시보드 상에서 즉시 프로젝트 초기화를 실행할 수 있는 원클릭 초기화 인터페이스를 제공함.
- **핵심 가설**: 진단 파일 부재 상태(`_no_diagnostics`)를 가상 카드로 리스트에 통합 노출하고, 프로젝트 미초기화 상태 감지 시 화면에 "Vibe Clinic 초기화하기" 버튼을 크게 노출하면 다른 프로젝트 로딩 시의 사용성 실패 지점이 완벽히 해소될 것이다.

## 2. User Review Required (피드백 요청 사항)

이번 근본 해결을 위해 적용할 두 가지 개선 설계에 대해 확인을 부탁드립니다.

### 설계 A. `_no_diagnostics` 가상 진단 노드의 리스트 렌더링 지원
진단 파일이 없을 때 `diagnosticsList`가 비어 있더라도, 백엔드 진단 결과(`results`)에 `_no_diagnostics`가 포함되어 있다면 이를 가상 진단 카드로 리스트에 생성하여 노출합니다.
- **표시 내용**: "No Diagnostics Found" (SYSTEM 레이어, WARNING 상태)
- **효과**: 상단 통계 바와 카드 리스트의 개수 불일치 현상이 완벽히 해결됩니다.

### 설계 B. 대시보드 내 원클릭 초기화(Initialize) 기능 추가
- **동작**: 현재 선택된 프로젝트 경로에 `.vibe-clinic` 폴더가 존재하지 않거나 진단 파일이 없을 때, 대시보드 카드 영역에 **[🩺 Vibe Clinic 초기화하기 (Initialize)]** 버튼을 노출합니다.
- **효과**: 대시보드 웹앱 내에서 다른 프로젝트를 가져온 즉시 진단 환경을 구성하고 가상 예제 진단을 생성하여 즉시 자가진단 사이클로 진입할 수 있게 합니다.
