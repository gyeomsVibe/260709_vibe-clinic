## 1. Opportunity Brief

### 해결하려는 문제

#### ① 경로 인식 오류 ("안 고쳐진다")
- **원인 1**: Windows PowerShell 5.1의 `Write-Host` 출력 인코딩이 기본 CP949(한국어 Windows)여서 한글 경로(`트래커_캘린더`)가 출력될 때 Node.js `execFile`의 기본 UTF-8 디코더와 충돌하여 `T??L_K????` 혹은 `TL_K`와 같은 깨진 문자로 바뀝니다.
- **원인 2**: `readBody(req)`에서 `data += chunk` 방식으로 버퍼를 결합해 UTF-8 멀티바이트 문자가 청크 경계에서 잘리거나 유실될 위험이 상존합니다.

#### ② AI 프로젝트 스캐너 무한 루프
- **원인**: 백엔드 `aiProvider.chat` 내의 `fetch()` 호출에 타임아웃이 설정되어 있지 않습니다. 프록시 환경, 방화벽 차단 또는 API 응답 지연 시 네트워크 소켓이 열려 있는 상태로 무한 대기(Hanging)하게 되어 대시보드에서는 스피너가 무한히 돌게 됩니다.

#### ③ 과도한 진단 캔버스 제거 및 공간 최적화 (목업 반영)
- **원인**: 마인드맵 형태의 연결선(SVG Bezier) 캔버스는 노드가 많아질 때 공간 낭비가 심하고 가시성이 저하됩니다. 사용자는 목업(Attachment 3)의 깔끔하고 정제된 카드 그리드 형태를 강력히 선호합니다.
- **의사 소견 비유 누락**: 코드 분석기 느낌의 차가운 용어(SYSTEM, TASK, OK, WARNING) 대신, 대시보드가 **인간을 치료하는 신뢰감 있는 의사**처럼 따뜻한 한글 의사 소견과 진단 용어로 상태를 알려주어야 합니다.

#### ④ 불필요한 BYOK 제공자 제거 (Gemini 전용)
- 어차피 Gemini만 사용할 것이므로 OpenAI, Anthropic 등의 선택지는 UI의 복잡도만 높입니다.

## 2. 해결 방안 (Go — 승인 대기)

### A. 인코딩 & 경로 근본 해결
1. **PowerShell 출력 인코딩 수정**: `folder-picker.ps1` 최상단에 `$OutputEncoding = [System.Text.Encoding]::UTF8` 및 `[Console]::OutputEncoding = [System.Text.Encoding]::UTF8`을 명시하여 강제로 UTF-8 출력을 하도록 수정합니다.
2. **`readBody(req)` 버퍼 축적 방식으로 교정**: 청크 데이터를 String 대신 Buffer 배열로 수집하여 `Buffer.concat().toString('utf8')`로 일괄 파싱하여 멀티바이트 유실을 원천 방지합니다.

### B. AI 스캐너 행 방지 (Fetch Timeout)
1. **AbortController 도입**: `aiProvider.chat` 내부의 fetch 호출에 12초 타임아웃을 적용합니다. 12초 이상 지연되면 자동으로 중단(Abort)되어 백엔드가 에러를 감지하고 안전하게 **로컬 스마트 스캐너** 분석 결과로 폴백합니다.

### C. Gemini 전용 및 UI 구조 오버홀
1. **BYOK 단순화**: 제공자 드롭다운을 삭제하고 오직 **Gemini API 설정**만 기입하도록 마크업을 단순화합니다.
2. **진단 캔버스 삭제 및 그리드화**: SVG 연결선과 3단 컬럼 계층을 전면 제거하고, 3열 반응형 `.cards-grid`로 교체합니다.
3. **의사-환자 치료 비유 한글화**:
   - `SYSTEM` -> `시스템 환경 검사`
   - `TASK` -> `핵심 업무 흐름 검사`
   - `FUNCTION` -> `개별 기능 단위 검사`
   - `OK` -> `완치 (정상 동작)` (에메랄드 그린 `#34D399` 테두리)
   - `WARNING` -> `주의 관찰 필요 (경고)` (앰버 `#FBBF24` 테두리)
   - `ERROR` -> `치료 시급 (오류 발견)` (로즈 `#FB7185` 테두리)
   - `PENDING` -> `진단 검사 대기` (회색 테두리)
   - 각 진단 카드 내에 **"🩺 의사 소견"** 구역을 신설하여, 상태별로 친절한 조치 사항과 증상을 표기합니다.
