## 1. 진단 요약 — 왜 "고쳤다"는데 계속 재발했는가

같은 증상("한글 경로 오류", "429 멈춤", "레이아웃")에 대한 완료 보고서가 **문서 24·25·27·29·31·33·35·37·38·40에 걸쳐 반복** 생산되었다. 원인은 세 가지 구조적 패턴이다.

| 패턴 | 내용 |
|---|---|
| ① 오진(誤診) | "한글 경로 오류" 하나의 증상 아래 실제로는 **서로 다른 3개의 버그**(인코딩, 미초기화 UX, ES 모듈 로딩)가 숨어 있었다. 한 번에 하나씩만 고쳐지니 사용자는 "안 고쳐진다"고 체감 |
| ② 검증 루프 붕괴 | 수정 자체가 자가검증 장치를 부쉈다. `folder-picker.ps1`의 `-DryRun` 자가검증이 param 버그로 죽은 상태에서 후속 수정이 반복됨 |
| ③ 기준선 부재 | 대량 미커밋 + 병렬 세션 경합으로 "어느 시점이 정상이었는지" 비교 기준(baseline)이 없어 회귀를 못 잡음 |

## 2. 고질 문제별 근본 원인 (실측 기준)

### 문제 A. 한글 경로 / 폴더 선택기 — ★ 핵심 잔존 버그 1건

**증상 이력**: 경로가 `T??L_K????`처럼 깨짐 → 7회에 걸쳐 "완료" 보고.

**계층별 원인과 현재 상태**:

| 계층 | 원인 | 현재 상태 |
|---|---|---|
| A-1 | PowerShell 5.1 CP949 출력 ↔ Node UTF-8 디코딩 충돌 | ✅ **Base64 터널링**(`SELECTED_B64:`)으로 실제 해결됨 (문서 37) |
| A-2 | A-1 1차 수정(인코딩 2줄)이 `param()` **앞에** 삽입됨 | ❌ **잔존**. PowerShell은 `param()`이 첫 실행문이어야 하므로 매 실행마다 `CommandNotFoundException` 발생, `-DryRun` 스위치 사망(실측 재현 완료). Base64 도입으로 이 2줄은 이제 **불필요한 죽은 코드**인데 버그만 남김 |
| A-3 | 미초기화 프로젝트가 건강도 0% + 빨간 에러로 표시되어 "경로 버그"로 오인 | ✅ 회색 `대기` 상태로 개편 (문서 38) |
| A-4 | `"type": "module"` 프로젝트에서 `.clinic.js` 로딩 실패 | ✅ `.cjs` 지원 + 로컬 처방 엔진 (문서 40) |

**단위 테스트가 못 잡는 이유**: 테스트는 `execFile`을 모킹하므로 실제 PowerShell 파싱을 한 번도 거치지 않는다.

### 문제 B. Gemini 429/타임아웃 무한 스피너 — 방어층 구축됨, 환경 제약 잔존

- 근본 원인: 무료 쿼터 한도 + 프로젝트 전환 시 중복 요약 호출 + fetch 무타임아웃.
- 현재: 25초 AbortController + 메모리 캐시 + 429/503/키 오류 시 즉시 로컬 폴백 + 로컬 처방 엔진까지 4중 방어 완료 (문서 35·38·40, 코드 실측 일치).
- 잔존 리스크: 429 자체는 코드로 제거 불가(환경 제약). 라이브 구동 시 실제로 폴백 배지가 떠 있었음 — **저장된 키 유효성/모델명(`gemini-3.5-flash`) 실검증은 미실시**.

### 문제 C. 수정 과정에서 새로 생긴 부채 (보안·부작용) — ★ 잔존 3건

| ID | 위치 | 내용 | 위험 |
|---|---|---|---|
| C-1 | `dashboard.js` `isAllowedDashboardOrigin` | `Origin: 'null'` 허용 — 어제 e01252f가 막은 구멍의 재개방. 로컬 `file://` HTML이 API에 POST 가능 | 높음 — C-2와 결합 시 코드 실행 경로 |
| C-2 | `POST /api/diagnostic/create` | `testCode` 필수/타입 검증 없음. 임의 코드가 `.clinic.js`로 저장되고 다음 진단 때 **실행됨** | 높음 |
| C-3 | `GET /api/errors` → `listErrorPatterns` | 패턴 1개 이하면 **어떤 프로젝트든** 계산기 전용 예제 2개를 GET 요청만으로 기록(쓰기 부작용). 이미 examples/calculator 오염, 타 프로젝트 전환 시에도 오염됨 | 중간 |

### 문제 D. 구성 관리 — 미커밋 드리프트 재확산

- 코드 9개 파일 수정 + 문서 32~40 미추적 상태. 테스트는 25/25 통과하나 커밋 기준선이 없어 다음 회귀 발생 시 원점 비교 불가.

## 3. 오류수정방안 (최소 수정 원칙: 1 failure → 1 cause → smallest fix → re-run)

### P0 — 즉시 (각 5분 이내, 되돌리기 쉬움)

**F-1. folder-picker.ps1 인코딩 2줄 삭제** (문제 A-2)
- 조치: 18~19행 `$OutputEncoding…` / `[Console]::OutputEncoding…` 삭제. Base64 출력은 순수 ASCII라 이 설정 자체가 불필요.
- 수용 기준: `powershell -NoProfile -STA -File src/folder-picker.ps1 -DryRun` → stderr 에러 0건 + `DRYRUN_OK` 출력 + 대화창 미표시.

**F-2. Origin `'null'` 허용 제거** (C-1)
- 조치: `if (origin === 'null') return true;` 1행 삭제. VS Code 웹뷰용 `vscode-webview://` 계열 허용은 유지.
- 수용 기준: `npm test` 전건 통과(Origin 차단 테스트 포함).

### P1 — 후속 (각 10분 이내)

**F-3. `/api/diagnostic/create` 입력 검증** (C-2)
- 조치: `typeof testCode !== 'string' || !testCode.trim()` 이면 400 반환. (id 정규식 검증은 기존 유지)

**F-4. 예제 오류 패턴 자동 생성 범위 제한** (C-3)
- 조치: `writeExampleErrorPatterns` 호출을 프로젝트 경로가 `examples/calculator`일 때로 한정하거나, GET에서 제거하고 `/api/project/init` 시점으로 이동. (권장: 후자 — GET은 읽기 전용이어야 함)

### P2 — 재발 방지 (승인 시)

**F-5. E2E 스모크 게이트 신설**: Windows에서 실제 PowerShell로 `-DryRun`을 실행해 `DRYRUN_OK`를 확인하는 테스트 1건을 `npm test`에 추가 — A-2형 회귀(모킹이 못 잡는 버그)의 영구 차단.

**F-6. 기준선 커밋**: F-1~F-4 반영 후 현재 작업분(코드/문서 분리)을 커밋해 비교 기준 확보. push는 별도 승인.

### P3 — 백로그

- Gemini 키·모델명 실검증(1회 실호출, 승인 필요) / README의 4개 프로바이더 표기 잔재 정리 / C5(잔액 0 WARNING 규칙) / 병렬 세션 동시 편집 방지 규칙(한 저장소 = 한 에이전트) 합의.
