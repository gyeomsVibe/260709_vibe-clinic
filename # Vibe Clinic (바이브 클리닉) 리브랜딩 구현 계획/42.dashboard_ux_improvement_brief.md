## 1. 개요 (Opportunity Brief)

### 1.1 배경 및 목적
현재 Vibe Clinic 자가진단 대시보드는 로컬 프로젝트의 코드 오류를 진단하고 AI를 통해 수리(Repair)하는 강력한 핵심 로직을 보유하고 있습니다. 그러나 현 UX/UI는 순수 HTML/CSS/Vanilla JS 아키텍처 하에서 기능 구동에 초점이 맞춰져 있어, 대규모 진단 항목 렌더링 시 **성능(Performance) 저하 우려**, **비정밀한 테마 시스템**, **시각적 정보 위계(Visual Hierarchy) 부족** 등의 한계를 가지고 있습니다. 
본 보고서는 대시보드를 정밀 분석하고 최신 웹 트렌드 리서치를 접목하여, 사용자 경험을 극대화(WOW)하고 구동 성능을 최적화하기 위한 구체적인 기획 및 개선 방향을 제시합니다.

### 1.2 핵심 이해관계자 및 사용자 여정 (User Journey)
*   **주요 사용자(윤겸스 님):** 로컬 코드베이스의 품질 상태를 시각적으로 모니터링하고, 에러 원인을 분석하며, AI 수리 패치를 안전하게 검토·적용하고자 하는 개발자.
*   **주요 Pain Points:**
    *   진단 항목 카드와 터미널 로그의 구분이 모호하여 가독성이 떨어짐.
    *   대량의 로그 렌더링 시 스크롤 버벅임 발생 우려.
    *   API 키 관리 및 환경 설정 영역이 한 영역에 결합되어 복잡하게 느껴짐.

3.  **마이크로 인터랙션 부재:** 버튼 클릭, 진단 프로세스 진행 시 흐름(Flow)이 딱딱하게 끊기며, AI 수리 제안 생성 시 "수리 제안 생성 중..."이라는 텍스트 외에 상세 진행 과정에 대한 로딩 상태가 시각적으로 체감되지 않습니다.
4.  **최적화 부재:** 진단 항목이 늘어나거나 대용량 터미널 stderr/stdout이 노출될 경우 DOM 노드가 급격히 증가하여 브라우저의 레이아웃 재계산(Reflow) 및 재페인팅(Repaint) 비용이 크게 발생합니다.

## 3. 웹 딥 리서치 기반 최적화 및 고품질 디자인 요소

### 3.1 디자인 요소: 다크 글래스모피즘(Dark Glassmorphism) & Bento Grid
1.  **공간적 아키텍처 (Spatial UI):** 
    *   단순한 평면 다크 모드가 아닌, 배경에 흐르는 유기적 그라데이션(`radial-gradient`) 위로 `backdrop-filter: blur(25px)` 처리된 미려한 투명 유리 카드 패널을 얹어 공간적 깊이(Depth)를 형성합니다.
    *   테두리에 아주 미세한 광원 효과(`border: 1px solid rgba(255,255,255,0.08)`)와 내부 그림자(`box-shadow`)를 부여하여 물리적인 유리판의 모서리를 표현합니다.
2.  **벤토 그리드 (Bento Grid):**
    *   복잡한 상태 정보를 크기와 배치가 비대칭적인 모듈형 그리드로 구획하여 직관성을 높입니다.
    *   예: 상단 핵심 지표(건강도, 진단 요약) -> 중단 메인 차트/노드 맵 -> 하단 치료 이력 및 로그 뷰어.

### 3.2 성능 최적화: 가상 스크롤(Virtual List) 및 렌더링 최적화
1.  **경량 가상 스크롤(Virtualization) 수립:**
    *   순수 Vanilla JS 환경을 감안하여 무거운 프레임워크 라이브러리 대신, **Intersection Observer API**를 이용해 뷰포트 영역 외부에 있는 진단 항목 노드는 화면에서 언마운트하거나 `content-visibility: auto` 속성을 적용해 브라우저 렌더링 큐를 비웁니다.
2.  **비동기 로그 스트리밍 및 Web Worker:**
    *   터미널 원시 로그 출력 시 수만 줄에 달하는 문자열을 한 번에 innerHTML로 주입하면 메인 스레드가 즉시 차단(Blocking)됩니다.
    *   대용량 텍스트 파싱 및 HTML 이스케이프 처리는 백그라운드 스레드인 **Web Worker**로 이관하고, 화면 출력은 `requestAnimationFrame`을 사용하여 스크롤 프레임을 유지합니다.
3.  **CSS 하드웨어 가속:**
    *   슬라이더, 드로어 오픈, 마이크로 반응 애니메이션 등의 동작에는 GPU 가속이 가능한 속성(`transform`, `opacity`)만 사용하고, `will-change`를 선언하여 레이아웃 리플로우(Reflow)를 원천 차단합니다.

## 4. 상세 개선안 및 변경 계획 (Decision Memo)

현재의 순수 Vanilla 아키텍처를 계승하여 가벼움을 유지할지, 아니면 확장성을 극대화하기 위해 최신 웹 프레임워크(React + Vite)로 리팩토링할지에 대해 두 가지 구현 대안을 제시합니다.

### 4.1 대안 비교 및 검토 (Candidate Portfolio)

| 평가 기준 | 옵션 A: Vanilla JS 고도화 (권장) | 옵션 B: React + Vite 아키텍처 전환 |
| :--- | :--- | :--- |
| **핵심 접근법** | 기존 HTML/CSS/JS 구조 유지하되 파일 분리, 스타일 고도화 및 최적화 로직 탑재 | Vite 번들러 도입, React 컴포넌트화 및 Tailwind/Shadcn UI 라이브러리 도입 |
| **구현 난이도** | **보통** (코드 구조 파악 완료 상태에서 모듈화 및 CSS 고도화 적용) | **높음** (Vite 설정, 빌드 파이프라인 변경 및 기존 라우터 API 포트폴리오 재구성 필요) |
| **성능 최적화** | Intersection Observer 기반 가상 스크롤 구축으로 가벼운 용량(Zero dependency) 유지 | react-window 등의 완성도 높은 라이브러리를 사용하여 안정적 가상화 구현 |
| **디자인 고도화** | 정밀하게 작성된 Vanilla CSS 변수(Tokens) 기반의 다크 글래스모피즘 수작업 튜닝 | Tailwind CSS의 유틸리티 클래스와 애니메이션 프리셋을 활용해 신속히 트렌디한 디자인 달성 |
| **배포 및 유지보수** | 빌드 과정 없이 `node` 명령어 단독으로 즉시 작동해 확장 프로그램 연동에 매우 유리 | 배포 전 번들링(Build) 단계가 필수적이므로 로컬 CLI 및 VS Code 확장 내부 구동 시 번들 배포 관리 필요 |
| **의사결정 게이트** | **Go (권장)**: 경량 자가진단 툴의 빌드 제로 철학 유지 및 즉각적 디자인 변신 가능 | **Research More**: CLI 구동 복잡도 증가 우려가 있으므로 장기 확장성 검토 시 적합 |

## 5. 단계별 실행 계획 (Roadmap)

    ```css
      --glass-panel: rgba(18, 18, 22, 0.7);
      --glass-border: rgba(255, 255, 255, 0.08);
      --glass-glow: rgba(56, 139, 253, 0.12);

      --accent-cyan: #06b6d4;
    }
    ```
*   Bento Grid 및 Glassmorphism 카드로 레이아웃 고도화.

### Phase 2: 성능 최적화 로직 적용 (Zero-Dependency)
*   **Intersection Observer 가상화 헬퍼** 도입:
    ```javascript
    const observer = new IntersectionObserver((entries) => {
      entries.forEach(entry => {
        if (entry.isIntersecting) {
          entry.target.classList.remove('dom-hidden');
        } else {
          entry.target.classList.add('dom-hidden');
        }
      });
    }, { rootMargin: '100px 0px' });
    ```
*   터미널 로그 스트리밍 시 대용량 데이터 Chunk 단위 쪼개기 렌더러 도입 (`requestIdleCallback` 또는 `setTimeout` 기반 스케줄링).

### Phase 3: 마이크로 인터랙션 및 UX 루프 강화
*   진단 실행 버튼 클릭 시 단순 회전 스피너를 넘어, 로딩 과정의 상태(스캔 중 -> 파일 분석 중 -> AI 의견 작성 중)를 순차 피드백하는 애니메이션 로더 추가.
*   수리 검토 모달의 코드 디프(Diff) 뷰어에 줄 번호 정렬 및 복사 기능, 구문 강조(Syntax Highlighting) 가독성 향상 패치.
